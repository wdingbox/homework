ACS5000 Configuration Importation Strategy

 

General

  1. The exported backup compressed config file (.tar.gz) from ACS5000 will be the primary data for migration to ACS8000.
  2. The exported backup config from ACS8000 will be the primary reference for target migration. (but may not limited to)
  3. The migration tool will be implemented in python. (codes)

User Account Migration

(Need reboot; cli script does not support migration.)

  1. /etc/passwd (linux file, hard copy, add: daemon:x:2:2:daemon:/var/empty:/bin/false)
  2. /etc/shadow (linux file, hard copy, add: daemon:*:-14671:0:99999:7:::)
  3. /etc/group (linux file, hard copy)
  4. /etc/portslave/pslave.conf => /etc/authorization.ini (avocent file, not in backup, update?)

Certifications

(Keep unchanged)

  1. .pem file keep unchanged.
  2. .key file keep unchanged.
  3. .pub file keep unchanged.

Network Configuration Files

  1. /etc/hostname (linux file, hard copy)
  2. /etc/hosts (linux file, copy)
  3. /etc/resolv.conf (linux file, copy)
  4. /etc/sysconfig/network (linux file, ???)
  5. /etc/sysconfig/network-scripts/ifcfg-interface-name (avocent file, ???)
